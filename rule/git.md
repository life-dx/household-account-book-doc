# Gitのルール

## 概要

本プロジェクトにおけるgit関連のルールです。

## ブランチと開発フロー

1. `feature/xxxxxx`というブランチを作成します。`xxxxxx`は機能名など、追加予定の項目がわかるような名前
2. 開発が完了したら、`main`ブランチへのプルリクエストを作成します。
3. レビューが完了したらマージします。

> `main`ブランチへの直接pushは禁止

## コミットルールについて

コミットメッセージにはprefixを付けてください。

| Prefix   | 用途                                   |
|----------|----------------------------------------|
| feat     | 新機能の追加                           |
| fix      | バグ修正                               |
| refactor | リファクタリング（機能変更なし）       |
| docs     | ドキュメントのみの変更                 |
| style    | フォーマット修正（空白、セミコロン等） |
| test     | テストの追加・修正                     |
| remove   | 削除                                   |
